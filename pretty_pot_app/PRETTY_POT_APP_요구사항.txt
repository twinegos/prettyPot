PRETTY_POT_APP 어플리케이션 요구사항

=== 1. 앱 기본 구조 ===
- 첫 화면: 터치하여 메인 메뉴로 진입
- 메인 메뉴: 3개 메뉴 (화병대여, 빈메뉴1, 빈메뉴2)
- 관리자 메뉴: 패스워드 입력으로 접근
- 데이터베이스: MongoDB 사용
- 플랫폼: 태블릿(매장용), 모바일(관리자용, 동기화)

=== 2. 화병대여 메뉴 구조 ===

2.1 고객 구분
- 신규 고객: 처음 대여하는 고객
- 기존 고객: 이미 대여중인 고객

2.2 신규 고객 프로세스
- 전화번호/주소 입력으로 등록
- 등록 과정 중 언제든 홈으로 돌아가기 버튼 제공
- 등록 완료 후 화병리스트(썸네일) 표시
- 화병 선택 → 동의 화면 → 고해상도 이미지와 가격 표시 → 완료 화면
- 완료 후 감사인사와 함께 자동으로 홈 화면 복귀 (일정시간 후)
- 즉시 홈으로 돌아가는 버튼도 제공

2.3 기존 고객 프로세스
- 전화번호/주소 입력으로 조회
- 현재 대여중인 화병리스트 표시
- 과거 대여 이력 표시
- 메뉴 선택: 반납 / 교체 또는 추가

=== 3. 반납 프로세스 ===
- 대여중인 화병리스트(썸네일) 표시
- 반납할 화병 선택
- 감사인사 및 대여료 반환 안내
- 홈으로 돌아가기 버튼
- 일정시간 후 자동 홈 복귀

=== 4. 교체/추가 프로세스 ===

4.1 1개 화병 대여중인 고객
- 전체 화병리스트(썸네일) 표시
- 원하는 화병 선택
- 교체/추가 중 선택
- 교체: 기존 화병과 새 화병 교체
- 추가: 기존 화병 + 새 화병 (구분 표시, 대여료 구분 표시)

4.2 2개 이상 화병 대여중인 고객
- 추가: 1개 대여중 고객의 추가 프로세스와 동일
- 교체: 
  * 현재 대여중인 화병리스트(썸네일) 표시
  * 추가/교체 선택
  * 교체 선택 시 교체할 대상 1개만 선택
  * 선택한 화병의 큰 사진으로 확인
  * 전체 화병리스트에서 새 화병 선택
  * 교체될 화병과 새 화병을 큰 사진으로 동시 표시하여 확인
  * 최종 확인 후 감사인사와 함께 홈 복귀

=== 5. 관리자 기능 ===
- 패스워드로 접근
- 화병리스트 관리 (썸네일, 고해상도 이미지)
- 동의서 내용 수정 기능
- 각 기능별 이미지 등록 및 관리

=== 6. 알림 기능 ===
- 교체, 반납, 추가 등의 작업 발생 시 관리자 휴대폰으로 SMS 전송

=== 7. 기술 요구사항 ===
- 데이터베이스: MongoDB
- 태블릿: 매장용 메인 시스템
- 모바일: 관리자용, 매장 태블릿과 동기화
- 자동 홈 복귀: 일정시간 후 자동으로 홈 화면으로 이동
- 즉시 홈 복귀: 사용자가 직접 홈으로 돌아갈 수 있는 버튼

=== 8. UI/UX 요구사항 ===
- 썸네일과 고해상도 이미지 구분 사용
- 직관적인 화면 구성
- 대여료 정보 명확히 표시 (기존/신규 구분)
- 감사인사 메시지 포함
- 확인 과정을 통한 사용자 실수 방지

=== 9. 빈 메뉴 ===
